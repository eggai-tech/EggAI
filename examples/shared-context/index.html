Context Sharing Between Agents with EggAI üß†üì°

Welcome to the 04-context example for EggAI! In this demonstration, you‚Äôll see how agents can share and utilize context across different stages of a user query without a dedicated context store. The context is now directly embedded and passed through the message events themselves. This approach streamlines the flow of information, reducing the reliance on external stores.

In this scenario, two agents collaborate:

  • Product Agent: Retrieves a list of relevant products based on a user query and includes the query context directly in its output message.
  • Recommendation Agent: Reads the context from the incoming message (which contains the user‚Äôs query and previously selected products) and uses it to suggest additional related items.

This pattern simulates a "You may also like..." flow often seen in e-commerce.

What‚Äôs Inside? üóÇÔ∏è

  • Product Agent:
  • Receives a user query (e.g., "smartphones").
  • Searches an in-memory database of products.
  • Returns a list of 3 best-matching products.

  • Embeds the query‚Äôs context (such as the search term and returned product IDs) directly into the response message.

  • Recommendation Agent:

  • Listens for product result events from the Product Agent.
  • Extracts context (query, product list) from the received message.
  • Suggests related items not in the initial product set, providing a richer and more coherent user experience.

By passing context directly in messages, we simplify the architecture and make the context chain more transparent and traceable.

Prerequisites üîß

Please ensure that you have the following installed:

  • Python 3.10+
  • Docker and Docker Compose
  • The EggAI framework (pip install eggai)

Architecture Overview üîÑ

The flow is as follows:

  1. User Query ‚Üí The Product Agent receives a search request (e.g., "I want a gaming smartphone, preferably Apple").
  2. Database Query ‚Üí The Product Agent fetches 3 matching products from the in-memory database.
  3. Direct Context Passing ‚Üí The Product Agent includes the original query and the chosen products‚Äô details in the same message that publishes the results.
  4. Recommendation Trigger ‚Üí The Recommendation Agent listens for the Product Agent's output message. Upon receipt, it extracts the context (the user query and returned products) directly from the message payload.
  5. Additional Suggestions ‚Üí Using the provided context, the Recommendation Agent recommends related items that complement the initial search results.
  6. Response to User ‚Üí The user receives both the initial product list and additional recommendations, demonstrating a context-rich user experience without external storage.

What to expect:

  2. Product Agent:
  3. Finds relevant products and returns three items (e.g., iPhone 15, Samsung Galaxy S23, OnePlus 11).
  4. The message it sends out includes the user's original query and these three product IDs in its payload.
  5. Recommendation Agent:
  6. Receives the Product Agent‚Äôs message, reads the embedded context, and identifies that the user prefers Apple and is interested in gaming.
  7. Suggests related items that were not included in the initial search results but match the user‚Äôs context (e.g., a high-performance Apple laptop or a gaming-focused laptop).
  8. Console Output: ```plaintext Agent is running. Press Ctrl+C to stop. User: Can you recommend a smartphone, I like gaming on it. I prefer Apple if possible Product Agent:
